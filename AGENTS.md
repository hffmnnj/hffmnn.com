## Commands (Auto)

All commands use `bun` as the runtime and package manager.

- **Install:** `bun install`
- **Dev server:** `bun run dev`
- **Type check:** `bun run check` — runs `svelte-kit sync && svelte-check --tsconfig ./tsconfig.json`
- **Build (static):** `bun run build` — adapter-static; writes to `build/`
- **Preview build:** `bun run preview`

> No test suite is configured. Verification is `bun run check` + `bun run build`.

## Gotchas (Auto)

- **`entries` generator is critical for static prerendering.** `adapter-static` silently skips any slug not returned by the `entries` function in `[slug]/+page.ts`. After adding a new project array, always spread it in `entries`. Verify with `ls build/tools/` after `bun run build`.
- **Tailwind v4 CSS custom properties require two registrations.** To add a new accent color (e.g., `bg-reins`): register `--{name}: oklch(...)` in `:root` AND map `--color-{name}: var(--{name})` in the `@theme inline` block of `app.css`. Missing the `@theme inline` entry means the utility class won't exist even if the CSS variable compiles.
- **`websiteUrl!` assertion required in Svelte templates.** When using `project.websiteUrl` inside an `{#if project.websiteUrl}` block, TypeScript still requires the non-null assertion (`project.websiteUrl!`). The narrowing doesn't propagate into Svelte template expressions.
- **LSP may show false positives on new exports.** After adding a new export to `projects.ts`, the Svelte language server may briefly report "has no exported member" even when `bun run check` passes clean. Trust `bun run check` as the source of truth.
- **404 on image paths in README.md files.** The `[slug]` detail pages fetch GitHub README.md content at build time. READMEs that reference relative image paths (e.g., `.github/assets/banner.png`) will produce 404 warnings in the build log. This is expected and harmless — the page renders fine with the 404 logged.
